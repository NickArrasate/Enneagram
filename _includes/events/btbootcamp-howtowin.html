What we'll cover:


When you launch a project on BowTie, you get a private github repo, SSL certificate, and DNS configuration all with the click of a button. No more waiting hours for the boring stuff to get set up. Set up your dev environment and install the BowTie client to get started.


Building

BowTie is push to deploy. Install the BowTie client to work locally. Many applications won't require a database. If you need to access/store data, checkout our static data files, or user profiles. You probably want to start by taking a look at your default settings.

We provide Jekyll for a fully configured markdown blog. Upload your images to the `/img` directory, embed videos, and use preconfigured components to take advanatge of services like addthis and disqus. Take a look at the `_includes` directory to see what's possible.

Styling

If you're focused on styling we recommend you hit up this user guide, and focus your energy on the `_sass` directory. This contains a `theme.scss` file loaded with variables to make styling your site really quick. You can also use bootstrap, jquery, and fontawesome, which are baked in by default.

Third Party Services

Does your app need payments? Stripe payments and subscription plans are baked in - and you can set policies to create paywalls with ease. Looking to extend BowTie even further? Use the BowTie Zapier integration to connect user events to hundreds of third-party services, like mailing list software, CRMs, and more.

Team Integration

Development is all about teamwork. With BowTie, you can add team members to your project to start collaborating instantly.
